  • Auction House - Java This project was developed as part of a class assignment and focused on building a simple marketplace simulation. The system included three main components: a bank, auction houses, and users. The bank managed all account balances, while auction houses could either set or auto-generate items for sale. Each user was given a bank account with a starting balance and could purchase items as long as they stayed within their budget. The design allowed for one centralized bank but unlimited users and auction houses. A bidding system ensured that users could only place bids if they had sufficient funds, adding a layer of financial validation to the process. For me, the most valuable part of this project was that it was my first experience coding a system that needed to connect across multiple devices. It challenged me to think about communication between different components and gave me a foundation in building interactive, multi-user systems.

The main learning objective of this project was to understand how to model complex systems and the physics behind celestial movements. I also learned how to use Haskell for simulations, which was a new experience for me.

  • Maze Solver - C I implemented a maze-solving program in C that explored different pathfinding algorithms, including Breadth-First Search (BFS), Depth-First Search (DFS), and the A* algorithm. The program was designed not only to find a way out of the maze but also to optimize traversal when multiple objectives, such as collecting items scattered throughout the maze, were introduced. Each algorithm brought different strengths and challenges. BFS guaranteed the shortest path in terms of steps, but could be memory-intensive for larger mazes. DFS offered a simpler approach but sometimes led to inefficient or longer paths. A*, by incorporating heuristics, balanced efficiency and accuracy, often producing the most optimized route when navigating toward both the exit and collectible items. Through this project, I gained hands-on experience with algorithm design and complexity analysis, while also deepening my understanding of how theoretical concepts like heuristics and search strategies apply to real-world problems. It was an exciting challenge to compare algorithm performance and see how their behaviors changed depending on the maze size, structure, and item placement.
  • Resturant POS - Java I began designing a restaurant point-of-sale (POS) system in Java, modeled after the SpeedLine system. The goal was to replicate core functionality such as order entry and menu navigation while building a flexible backend that could support a user-friendly interface. Early in development, I focused heavily on structuring the backend properly. I learned how to create reusable methods that could dynamically generate the buttons and interface elements needed for different menu items. This approach meant I wouldn’t have to hardcode each button, making the system more scalable and easier to maintain. Once I had successfully built the reusable backend framework, I made the decision to stop development before completing the full interface. Even though the project wasn’t finished, it was a valuable milestone in my learning process — it taught me the importance of writing clean, modular code and how thoughtful design early on can make future development much more efficient.
  • Tally Counter - Swift This project was a simple tally counter app developed in Swift for iOS devices. The app allowed users to increment or decrement a counter value with the tap of a button. It also included features to reset the counter and save the counter value for future reference. The user interface was designed to be clean and intuitive, making it easy for users to quickly access the controls and see the current count. The main learning objective of this project was to get familiar with Swift and iOS app development. I had always wanted to learn how to create mobile apps, and this project provided a great introduction to the basics of app design and functionality.
